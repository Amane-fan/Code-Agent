# 安全模型

## 默认保护

- 上下文收集会跳过常见依赖目录、构建目录、缓存目录、二进制文件和 VCS 目录。
- `.env`、`.npmrc`、`.pypirc`、私钥和凭据文件等敏感名称不会被读取。
- 疑似密钥的值会在工具输出展示或存储前被脱敏。
- Shell 命令默认必须匹配安全前缀，除非用户显式传入 `--unsafe`。
- 补丁应用前必须先通过 `git apply --check` 校验。

## 使用者责任

- 使用 `--apply` 前应审查模型生成的 diff。
- 除非命令可信，否则避免使用 `--unsafe`。
- 不要把 `.code-agent/` 提交到版本控制中，因为它可能包含任务 prompt 和运行历史。
- 为模型 Provider 使用最小权限 API key。

## 后续加固方向

- 在交互模式下加入按工具审批。
- 即使用户给了宽松权限，也对破坏性操作加入命令拒绝列表。
- 增加工作区写入策略，将可编辑范围限制在指定目录。
- 自动检测并阻止触碰密钥文件的生成补丁。
